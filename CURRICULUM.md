# Curriculum blueprint — Tu Viaje Matemático

The honest bridge from counting to calculus. This is the plan the app's `SKILLS`
graph should grow into.

## Principles

1. **Each tree (skill) is novel** — it introduces one genuinely new idea.
2. **Inside a tree, many lessons** — small steps that reinforce and drill the same
   idea until it's solid. Depth over breadth per concept.
3. **No hidden prerequisites** — every micro-skill a harder problem needs gets its
   own lesson *before* it's needed (factoring, exponent laws, like terms, sign
   rules, common denominators…).
4. **Prerequisites gate the map** — a skill unlocks only when its prereqs are mastered.
5. **Correctness is non-negotiable** — every answer is verified by running code
   (SymPy / arithmetic), never by hand.

Legend: ✅ exists today · 🔨 exists but must be deepened · 🆕 new skill · 🚧 future

## Strand 1 — Números y aritmética (Fundamentos)

- 🔨 **Números y valor posicional** — contar y la recta numérica · unidades/decenas/centenas · miles y millones (leer y escribir) · comparar y ordenar · redondear
- 🔨 **Suma** — sin llevar · llevando (reagrupar) · números grandes · estrategias mentales · problemas
- 🆕 **Resta** — sin pedir prestado · pidiendo prestado · restar con ceros (100 − 37) · problemas
- 🔨 **Multiplicación** — suma repetida · tablas · conmutativa/asociativa/distributiva · ×10, ×100 · varias cifras · problemas
- 🆕 **División** — repartir / inverso de multiplicar · división básica · división larga · con resto · ÷10, ÷100 · problemas
- 🆕 **Factores, múltiplos y primos** ⟵ *su punto flojo* — factores de un número · múltiplos · reglas de divisibilidad (2,3,5,9,10) · primos y compuestos · factorización en primos (árbol) · MCD · mcm
- 🔨 **Orden de las operaciones** — por qué hay una jerarquía · paréntesis primero · × y ÷ (izq→der) · + y − (izq→der) · con potencias · expresiones anidadas

## Build order

**Depth-first, bottom-up.** Fully build and code-verify Strand 1, then 2, 3…
Each strand ships as soon as its lessons are written and its answers pass the
verifier. Upper strands stay visible as stubs so the whole plan is always on the
map — she can see calculus is coming, even before it's built.
